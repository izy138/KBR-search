Frontend
--------

The UI is a React 19 + TypeScript app built with Vite. All data comes from the FastAPI backend via frontend/src/api.ts.

Base URL: VITE_API_BASE_URL if set, otherwise http://localhost:8000.

Routes and pages
----------------

Routing is defined in frontend/src/App.tsx using react-router-dom.

URL	Component	What the user sees
`/`	`App` (search or dashboard tab)	Main search with filters, or analytics dashboard
`/projects/:projectId`	`ProjectDetailsPage`	Single grant: metadata, abstract, similar projects, other fiscal years
`/investigators/:investigatorName`	`InvestigatorPage`	All grants for one PI (name is URL-encoded)
`/semantic`	`SemanticVectorLabPage`	Text similarity, hybrid search, link to project-based similarity
`/semantic/similar/:projectId`	`SemanticSimilarProjectPage`	Projects semantically similar to a chosen grant

The header switches between Search, Dashboard, and Vector lab without leaving the single-page app shell.

Main views
----------

Search (`/` — search tab)
-------------------------

Files: SearchBar.tsx, Filters.tsx, ResultsList.tsx, App.tsx

- User enters a query; searchProjects() calls GET /search/.
- Filters (PI, IC, activity, state, fiscal year) are sent as query parameters.
- Pagination uses page and limit (default 25 per page).
- Clicking a result navigates to /projects/{id}.
- Filter dropdown options are derived from the current result set.

Dashboard (`/` — dashboard tab)
-------------------------------

File: components/Dashboard.tsx

Loads multiple analytics endpoints in parallel and feeds chart/map components:

UI area	API helper	Endpoint
Summary stats	`getDashboardSummary`	`GET /analytics/summary`
U.S. state map	`getStateData`	`GET /analytics/by-state`
IC bar chart	`getIcData`	`GET /analytics/by-ic`
Activity bars	`getActivityData`	`GET /analytics/by-activity`
Funding pie	`getActivityFundingPie`	`GET /analytics/by-activity-funding-pie`
Term theme cloud	`getProjectTermThemeCloud`	`GET /analytics/project-term-theme-cloud`
Funding over time	`getYearData`	`GET /analytics/by-year`
Top organizations	`getTopOrgs`	`GET /analytics/top-orgs`
Avg grant by IC	`getAvgGrantByIc`	`GET /analytics/avg-grant-by-ic`

StateMap.tsx does not call the API; it receives state aggregates from Dashboard.

Project details (`/projects/:projectId`)
----------------------------------------

File: components/ProjectDetailsPage.tsx

Action	API	Endpoint
Load project (in `App.tsx`)	`getProjectById`	`GET /search/project/{id}`
Similar grants	`searchSimilarToProjectId`	`GET /search/similar/{id}`
Other fiscal years	`getProjectOtherYears`	`GET /search/project/{id}/other-years`

ProjectActivityTermsChart.tsx on this page calls GET /analytics/by-activity-project-compare to compare the project with peers in the same activity code.

Investigator page (`/investigators/:name`)
------------------------------------------

Files: InvestigatorPage.tsx, App.tsx

getProjectsByInvestigator() → GET /search/investigator/{pi_name} with pagination.

Vector lab (`/semantic`)
------------------------

File: components/SemanticVectorLabPage.tsx

Mode	API	Endpoint
Keyword search (baseline)	`searchProjects`	`GET /search/`
Semantic similarity	`searchSimilarByText`	`GET /search/similar`
Hybrid (keyword + vector)	`searchHybrid`	`GET /search/hybrid`

Project-based similarity uses SemanticSimilarProjectPage and GET /search/similar/{project_id}.

Shared presentation components
------------------------------

These components only render props; they do not fetch data:

- BarChartPanel.tsx, LineChartPanel.tsx — Recharts bar/line charts
- ActivityFundingPiePanel.tsx — pie chart for activity funding
- ProjectTermsThemeCloud.tsx — word cloud from precomputed theme JSON
- VerticalOnlyBarShape.tsx — custom bar shape for IC chart
- Charts.tsx — legacy/simple chart wrapper

API helpers not wired in the UI
-------------------------------

Defined in api.ts but unused by components today:

- getHealth() → GET /health
- getAnalyticsSummary() → GET /analytics/summary (dashboard uses the richer summary helper)
- getActivityTermsData() → GET /analytics/by-activity-terms
